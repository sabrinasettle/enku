---
name: feedback_ask_before_removing
description: Always ask before removing or replacing existing design elements — even if the replacement seems better
metadata:
  type: feedback
---

Do not remove existing design elements without asking first. If something seems wrong or could be improved, ask what the user wants rather than silently deleting/replacing it.

**Why:** User explicitly corrected this when OutfitsView text was removed and replaced with images. The position text was intentional and paired with a visual infographic concept the user had in mind.

**How to apply:** When a feature or element seems redundant, unclear, or improvable, ask "do you still want X or should I replace it with Y?" before making the change.
